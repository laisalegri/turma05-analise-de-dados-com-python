1. Versionamento com Git e GitHub

Na Semana 01 você já criou sua conta no GitHub, sua branch e seu primeiro Pull Request — mas seguiu o passo a passo sem ainda entender a fundo cada conceito. Agora é hora de entender o que cada um desses termos realmente significa.

ConceitoO que é
CommitUm "ponto de salvamento" das suas mudanças, com uma mensagem explicando o que foi feito
BranchUma cópia paralela do repositório, só sua, onde você trabalha sem afetar os colegas
MergeJuntar as mudanças de uma branch de volta na branch principal
Pull Request (PR)O pedido formal para alguém revisar e aprovar sua mudança antes dela entrar na main

O fluxo simplificado (GitFlow)

1. Criar branch
→
2. Commits
→
3. Push
→
4. Pull Request
→
5. Merge na main
💡 Por que não commitar direto na main? Porque qualquer erro ficaria imediatamente na versão oficial do projeto, visível para todo mundo. Trabalhar em branch + revisão via PR é o que protege o trabalho de toda a equipe.

Comando essencial: o commit

git add arquivo.py
git commit -m "Adiciona função de cálculo do total do carrinho"
✏️ Agora é sua vez: no seu computador (ver tutorial de instalação na próxima seção), crie um repositório de teste e pratique esses dois comandos.

O merge na prática (e o que é um conflito)

"Merge" não é só uma palavra no fluxograma acima — é um comando real, e é o momento em que o Git tenta juntar automaticamente as mudanças de duas branches:

git checkout main
git merge pratica-semana-03

Na maioria das vezes isso funciona sozinho, sem você precisar fazer nada. Mas se duas pessoas alteraram a mesma linha do mesmo arquivo de jeitos diferentes, o Git não sabe qual versão manter — isso é um conflito de merge, e ele marca o trecho problemático direto no arquivo, assim:

<<<<<<< HEAD
total = preco * quantidade
=======
total = preco * quantidade * 1.1  # com imposto
>>>>>>> pratica-semana-03
💡 Conflito não é um erro seu — é o Git pedindo ajuda. Você escolhe manualmente qual trecho fica (ou escreve um terceiro, combinando os dois), apaga as linhas com <<<<<<</=======/>>>>>>>, e faz um novo commit. É exatamente por isso que branches pequenas e commits frequentes (em vez de um commit gigante no final) tornam conflitos mais raros e mais fáceis de resolver.

O que o Git escolhe ignorar: .gitignore

Nem tudo numa pasta de projeto deve ser versionado — senhas, arquivos temporários do sistema, pastas geradas automaticamente. O arquivo .gitignore lista, linha a linha, o que o Git deve ignorar e nunca registrar em nenhum commit:

# .gitignore
*.pyc
__pycache__/
.env

É exatamente esse mecanismo que mantém os arquivos internos do professor (planos de aula, gabaritos) fora do repositório público da turma.

2. Instalando Python e VS Code

Até agora você só programou dentro do Google Colab, que já vem com Python pronto no navegador. A partir de hoje, você também vai programar localmente, no seu próprio computador — é assim que profissionais de dados trabalham no dia a dia.

Os exemplos e atividades de listas/tuplas/dicionários da seção 3 e o treino em squads continuam no notebook_colab.ipynb — é mais fácil para todo mundo da turma (e do squad) ver o mesmo código ao mesmo tempo. O ambiente local que você instala agora é usado no Exercício E9 (`04_exercicios.md`) e cresce de protagonismo a partir da Semana 04, quando você vai escrever e organizar scripts `.py` de verdade, fora do navegador.

Passo a passo — Ambiente local

  1. Instalar o Python

    O que é: o "motor" que executa qualquer código Python no seu computador — sem ele instalado, nenhum arquivo .py roda fora do navegador.

    Como fazer (Windows): acesse python.org/downloads, baixe a versão mais recente, abra o instalador e marque a opção "Add Python to PATH" antes de clicar em Install — esse passo é o que mais gera erro quando esquecido.

    Como fazer (Mac): acesse python.org/downloads e baixe o instalador para macOS (ou, se já usa Homebrew, rode brew install python no Terminal). O Mac já resolve o PATH automaticamente — não existe a etapa "Add Python to PATH" por lá.

  2. Instalar o VS Code

    O que é: um editor de código gratuito, leve e muito usado no mercado — é onde você vai escrever e executar seus arquivos .py localmente.

    Como fazer: acesse code.visualstudio.com, baixe para o seu sistema e siga as opções padrão do instalador.

  3. Instalar a extensão Python no VS Code

    Por quê: sem essa extensão, o VS Code não entende a sintaxe Python nem oferece o botão de "Executar" — ele trataria o arquivo como texto simples.

    Como fazer: abra o VS Code → ícone de extensões na barra lateral → busque "Python" (da Microsoft) → Install.

  4. Criar e executar seu primeiro script local

    A diferença do Colab: no Colab você executa célula por célula; localmente, você executa o arquivo inteiro de uma vez, pelo terminal.

    Como fazer: no VS Code, File → New File → salve como meu_primeiro_script.py → escreva print("Rodando localmente!") → clique no botão ▶ (Run) no canto superior direito.

    print("Rodando localmente!")

3. Listas, Tuplas e Dicionários

Voltando ao Python (seção 2 foi sobre ferramentas, não sobre código): até a Semana 02, cada variável guardava um único valor. Agora você aprende a guardar vários valores organizados em uma única estrutura.

EstruturaCaracterísticaExemplo
ListaMutável, ordenada, acessada por posiçãocompras = ["arroz", "feijão"]
TuplaImutável (não pode mudar depois de criada)coordenadas = (10, 20)
DicionárioAcessado por chave (nome), não por posiçãoproduto = {"nome": "Mouse", "preco": 45}

Exemplo 1 (resolvido) — Lista de compras

compras = ["arroz", "feijão", "leite"]

print(compras[0])     # primeiro item
print(compras[-1])    # último item

compras.append("café") # lista é mutável: posso adicionar
print(compras)
⚠️ Veja como é um erro real do Python: a lista compras tem 4 itens (índices 0 a 3). Se você pedir um índice que não existe, como compras[10], o Python mostra: 
IndexError: list index out of range
"Índice fora do intervalo" — é o Python dizendo que você pediu uma posição que a lista não tem. Tente provocar esse erro de propósito no notebook.

Exemplo 2 (resolvido) — Tupla: dados que não devem mudar

coordenadas = (10, 20)

print(f"X = {coordenadas[0]}, Y = {coordenadas[1]}")
⚠️ Veja como é um erro real do Python: se você tentar alterar um item de uma tupla, como em coordenadas[0] = 99, o Python recusa e mostra: 
TypeError: 'tuple' object does not support item assignment
"O objeto tupla não permite atribuição de item" — é exatamente a imutabilidade da tupla sendo aplicada. Tente provocar esse erro de propósito no notebook.

Exemplo 3 (resolvido) — Dicionário de produto

produto = {"nome": "Mouse", "preco": 45.0, "estoque": 12}

print(f"{produto['nome']} custa R$ {produto['preco']}")
print(f"Estoque disponível: {produto['estoque']} unidades")
✏️ Atividade Prática (no notebook): crie uma lista de dicionários representando um pequeno carrinho de compras e calcule o valor total.
