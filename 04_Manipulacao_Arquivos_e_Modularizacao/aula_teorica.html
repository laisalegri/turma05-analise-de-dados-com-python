1. Arquivos: CSV e JSON

Até agora, todo dado que você usou existia só enquanto o programa rodava. Arquivos permitem guardar e recuperar dados entre uma execução e outra.

FormatoUso comumMódulo Python
CSVPlanilhas simples, exportação de sistemascsv
JSONConfigurações, dados de APIs webjson
ExcelPlanilhas com formatação e múltiplas abasopenpyxl (ou Pandas, a partir do Bloco 3)
💡 Como escolher entre CSV e JSON? Use CSV quando o dado é uma tabela simples, sempre com as mesmas colunas (ex.: lista de alunos com nome/nota). Use JSON quando os dados têm estrutura mais "aninhada" ou variável (ex.: um produto que às vezes tem desconto, às vezes não) — é o mesmo formato que a maioria das APIs web devolve.

Exemplo 1 (resolvido) — Escrevendo um arquivo JSON

import json

produto = {"nome": "Mouse", "preco": 45.0, "estoque": 12}

with open("produto.json", "w", encoding="utf-8") as arquivo:
    json.dump(produto, arquivo, ensure_ascii=False)

print("Arquivo salvo com sucesso!")

Exemplo 2 (resolvido) — Lendo um arquivo JSON

import json

with open("produto.json", "r", encoding="utf-8") as arquivo:
    dados = json.load(arquivo)

print(f"{dados['nome']} custa R$ {dados['preco']}")
✏️ Agora é sua vez: no notebook, escreva uma lista de dicionários (alunos) em um arquivo CSV e leia esse arquivo de volta.
⚠️ Veja como é um erro real do Python: se você tentar abrir um arquivo que não existe, como open("nao_existe.json", "r"), o Python recusa e mostra: 
FileNotFoundError: [Errno 2] No such file or directory: 'nao_existe.json'
É o erro mais comum de quem trabalha com arquivos — geralmente é só um nome digitado errado ou um arquivo que ainda não foi criado. Você pode se proteger dele com try/except: 
try:
    with open("nao_existe.json", "r", encoding="utf-8") as arquivo:
        dados = json.load(arquivo)
except FileNotFoundError:
    print("Arquivo não encontrado — verifique o nome ou crie o arquivo primeiro.")

Bônus: Excel com openpyxl

O Colab já vem com openpyxl instalado. A lógica é parecida com CSV/JSON: abrir, ler/escrever, fechar.

from openpyxl import Workbook, load_workbook

# escrevendo
planilha = Workbook()
aba = planilha.active
aba.append(["nome", "preco"])
aba.append(["Mouse", 45.0])
planilha.save("produtos.xlsx")

# lendo de volta
planilha = load_workbook("produtos.xlsx")
aba = planilha.active
for linha in aba.iter_rows(values_only=True):
    print(linha)

A partir do Bloco 3, você vai preferir pandas.read_excel() para isso — mas é bom saber que existe a forma "pura" do Python também.

2. Datas e Expressões Regulares

Datas com datetime

from datetime import date

hoje = date.today()
fim_do_curso = date(2026, 8, 14)

dias_restantes = (fim_do_curso - hoje).days
print(f"Faltam {dias_restantes} dias para o fim do curso.")

Expressões regulares com re

Uma expressão regular (regex) é um padrão de texto usado para validar ou buscar formatos específicos — como e-mails, telefones ou CPFs.

import re

email = "aluno@exemplo.com"
padrao = r"^[^\s@]+@[^\s@]+\.[^\s@]+$"

if re.match(padrao, email):
    print("E-mail parece válido!")
else:
    print("E-mail inválido.")
Trecho do padrãoO que verifica
^Início do texto
[^\s@]+1 ou mais caracteres que não sejam espaço (\s) nem @ — a parte antes do @
@O @ obrigatório, exatamente uma vez
[^\s@]+De novo: 1+ caracteres sem espaço/@ — o domínio
\.Um ponto literal (o \ evita que . signifique "qualquer caractere")
[^\s@]+A extensão do domínio (ex.: com)
$Fim do texto
⚠️ Cuidado com padrões "soltos": um padrão mais simples como r".+@.+\..+" (sem ^/$ e sem excluir espaços) parece funcionar, mas na prática aceita coisas erradas, porque . sozinho significa "qualquer caractere" — inclusive espaço e até outro @. Teste sempre com um caso que deveria falhar (ex.: "a b@c d.e"), não só com um caso válido.
💡 Você não precisa decorar regex de memória — no mercado, a maioria dos profissionais consulta ou usa ferramentas para gerar o padrão certo. O importante é entender o que cada trecho do padrão verifica, como na tabela acima.

3. Funções: parâmetros, valores padrão e return

Olhe este código que calcula o total de 3 clientes diferentes:

total_ana = 10 * 2 - 0
total_bruno = 25 * 1 - 5
total_carla = 8 * 4 - 2

print(total_ana, total_bruno, total_carla)
⚠️ O que tem de errado aqui? A mesma conta (preço × quantidade − desconto) está escrita 3 vezes. Se um dia essa regra mudar (ex.: incluir um imposto), você precisaria encontrar e corrigir as 3 linhas — e é fácil esquecer uma. Uma função resolve exatamente isso: você escreve a regra uma única vez e só chama ela: 
def calcular_total(preco, quantidade, desconto=0):
    return preco * quantidade - desconto

print(calcular_total(10, 2))        # sem desconto: 20
print(calcular_total(10, 2, 5))     # com desconto: 15
💡 "Sem return, a função é surda — não responde nada." Uma função sem return pode até imprimir algo na tela, mas não devolve nenhum valor que você possa guardar em uma variável ou usar em outro cálculo.

Exemplo 1 (resolvido) — Função com retorno

def media(lista):
    return sum(lista) / len(lista)

notas = [7.5, 8.0, 6.5]
print(media(notas))

Exemplo 2 (resolvido) — Parâmetro opcional

def saudacao(nome, formal=False):
    if formal:
        return f"Bom dia, {nome}."
    return f"Oi, {nome}!"

print(saudacao("Ana"))
print(saudacao("Ana", formal=True))
✏️ Agora é sua vez: no notebook, crie a função calcular_total e teste com e sem desconto.
🚀 Avançado (opcional) — parâmetros "ilimitados" com *args: toda função até aqui tinha um número fixo de parâmetros. Com *args, uma função aceita qualquer quantidade de argumentos, que chegam agrupados numa tupla: 
def media(*args):
    return sum(args) / len(args)

print(media(7, 8, 9))        # funciona com 3 valores
print(media(7, 8, 9, 10))    # e também com 4, sem mudar a função
Isso é a base do desafio opcional do notebook desta semana.

4. Lambda e Módulos

Exemplo 1 (resolvido) — Função lambda (função de uma linha)

media = lambda lista: sum(lista) / len(lista)

print(media([7.5, 8.0, 6.5]))

Uma lambda é equivalente a uma função criada com def, só que escrita em uma única linha — útil para funções pequenas e simples.

Exemplo 2 (resolvido) — Onde o lambda realmente brilha: dentro de sorted()

Guardar uma lambda numa variável (como acima) é raro na prática. O uso real e mais comum é passar a lambda direto como argumento de outra função, como sorted(), max() ou min(), para dizer "ordene/compare por este critério":

produtos = [
    {"nome": "Mouse", "preco": 45.0},
    {"nome": "Teclado", "preco": 89.0},
    {"nome": "Borracha", "preco": 2.0},
]

mais_barato = min(produtos, key=lambda p: p["preco"])
print(f"O mais barato é {mais_barato['nome']}, por R$ {mais_barato['preco']}")
💡 Como ler isso: key=lambda p: p["preco"] diz ao min() "para decidir qual item é o menor, olhe o preco de cada um (p)". Sem a lambda, você precisaria escrever um for manual comparando item a item — a lambda só descreve o critério, e o Python faz a comparação para você.

Criando seu próprio módulo

Um módulo é só um arquivo .py com funções, que pode ser importado em qualquer outro script — assim você nunca precisa reescrever a mesma função duas vezes.

# arquivo: utilidades.py
def media(lista):
    return sum(lista) / len(lista)

def calcular_total(preco, quantidade, desconto=0):
    return preco * quantidade - desconto
# outro arquivo, na mesma pasta
import utilidades

print(utilidades.media([7.5, 8.0, 6.5]))
print(utilidades.calcular_total(10, 2, 5))
✏️ Agora é sua vez: crie seu próprio utilidades.py com pelo menos 3 funções e importe-o em outro script.
