1. Pandas: Series e DataFrame

EstruturaO que é
SeriesUma única coluna de dados, com índice
DataFrameUma tabela completa, com várias colunas (várias Series juntas)

Exemplo 1 — Criando um DataFrame

import pandas as pd

vendas = pd.DataFrame({
    "produto": ["Caderno", "Lápis", "Mochila", "Borracha"],
    "categoria": ["Papelaria", "Papelaria", "Acessórios", "Papelaria"],
    "preco": [12.5, 2.0, 150.0, 1.5],
    "quantidade_vendida": [30, 80, 5, 60],
})

print(vendas.head())
print(vendas.shape)

Exemplo 2 — Series x DataFrame

precos_series = vendas["preco"]          # Series
precos_dataframe = vendas[["preco"]]     # DataFrame (mantém a estrutura de tabela)

print(type(precos_series))
print(type(precos_dataframe))
💡 Colchete duplo = mantém a estrutura de tabela. É o erro mais comum desta semana: df['col'] vira Series, df[['col']] continua DataFrame.

Exemplo 3 — Exportando e lendo um arquivo CSV real

vendas.to_csv("vendas.csv", index=False)

vendas_lida = pd.read_csv("vendas.csv")
print(vendas_lida.head())
💡 É assim que você vai carregar qualquer dataset real (do Kaggle, de um relatório da empresa, de um portal de dados abertos): com pd.read_csv("nome_do_arquivo.csv"). O DataFrame vendas deste exemplo nasceu de um dicionário só para fins didáticos — mas a partir de agora, todo dataset que você receber vai chegar como um arquivo, e a leitura é sempre essa mesma linha de código.

2. Explorando um DataFrame: describe, filtros e groupby

Exemplo 1 — describe()

print(vendas.describe())

describe() é como rodar uma análise estatística instantânea: média, mínimo, máximo, desvio padrão — tudo de uma vez.

⚠️ Veja como é um erro real do Python: se você digitar o nome de uma coluna errada, como vendas["preco_errado"], o Pandas recusa e mostra: 
KeyError: 'preco_errado'
Antes de selecionar uma coluna, confira os nomes exatos com vendas.columns.

Selecionando por posição ou por rótulo: .loc e .iloc

print(vendas.loc[0, "produto"])      # .loc: pelo nome da coluna + posição da linha
print(vendas.iloc[0, 0])             # .iloc: só por posição numérica (linha 0, coluna 0)
💡 .loc usa rótulos (nomes), .iloc usa posições (números). Os dois fazem a mesma coisa de jeitos diferentes — use .loc quando você sabe o nome da coluna, e .iloc quando só sabe a posição (ex.: "a primeira linha, a primeira coluna").

Exemplo 2 — Filtro e groupby

# Filtro: só produtos vendidos mais de 20 vezes
mais_vendidos = vendas[vendas["quantidade_vendida"] > 20]
print(mais_vendidos)

# groupby: preço médio por categoria
print(vendas.groupby("categoria")["preco"].mean())

Como o groupby pensa por dentro: Dividir → Aplicar → Combinar

EtapaO que acontece
1. Dividir (split)O Pandas separa as linhas em grupos, um para cada valor de categoria — todas as linhas "Papelaria" em um grupo, "Acessórios" em outro
2. Aplicar (apply)Dentro de cada grupo separado, calcula a função pedida (ex.: mean() dos preços apenas daquele grupo)
3. Combinar (combine)Junta o resultado de cada grupo numa única Series/tabela de resposta
💡 groupby é o mesmo que uma Tabela Dinâmica no Excel: é exatamente isso que ela faz por trás dos panos — o groupby só está explicitando os 3 passos da tabela acima.

Ordenando o resultado: sort_values()

preco_medio_por_categoria = vendas.groupby("categoria")["preco"].mean()
print(preco_medio_por_categoria.sort_values(ascending=False))

sort_values() ordena os valores do maior para o menor (ou o contrário) — útil para responder "qual categoria tem o maior preço médio?" sem precisar procurar manualmente o maior número na tabela.

3. NumPy: arrays e operações vetorizadas

Exemplo 1 — Array x lista

import numpy as np

precos = np.array([12.5, 2.0, 150.0, 1.5])
quantidades = np.array([30, 80, 5, 60])

receita = precos * quantidades   # multiplicação elemento a elemento, sem for!
print(receita)
⚠️ Veja como é um erro real do Python: tente fazer a mesma multiplicação com listas comuns (não arrays): 
lista1 = [12.5, 2.0, 150.0]
lista2 = [30, 80, 5]
lista1 * lista2
TypeError: can't multiply sequence by non-int of type 'list'
É exatamente por isso que o NumPy existe: arrays permitem multiplicação elemento a elemento, listas comuns não.

Exemplo 2 — Broadcasting (escalar com array)

precos_com_aumento = precos * 1.10   # aumenta 10% em todos de uma vez
print(precos_com_aumento)

print(f"Receita total: {receita.sum()}")
print(f"Preço médio: {precos.mean()}")

Broadcasting é a capacidade do NumPy de operar um array com um único número (ou com outro array de formato compatível) sem precisar repetir a operação manualmente.

⚠️ Quando o broadcasting NÃO funciona: arrays de tamanhos incompatíveis também geram um erro real, não um resultado "estranho": 
a = np.array([1, 2, 3])
b = np.array([1, 2])
a * b
ValueError: operands could not be broadcast together with shapes (3,) (2,)
Os dois arrays precisam ter o mesmo tamanho (ou um deles ser um único número/escalar).
