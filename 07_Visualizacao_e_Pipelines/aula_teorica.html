1. Matplotlib: Gráficos Básicos

Depois de limpar os dados na Semana 06, chegou a hora de mostrá-los. Todo gráfico em Matplotlib começa com a mesma receita:

import matplotlib.pyplot as plt

fig, ax = plt.subplots()   # fig = a "folha"; ax = os "eixos" onde o gráfico é desenhado
ax.plot(...)                # ou ax.bar(...), ax.scatter(...)
plt.show()
💡 Por que fig, ax = plt.subplots() desde o início? Existe um jeito mais curto (plt.plot(...) direto), mas ele só funciona bem para 1 gráfico isolado. O padrão fig, ax é o que você vai usar sempre — inclusive quando tiver vários gráficos juntos (Seção 2) — então já aprenda do jeito que escala.

Exemplo 1 (resolvido) — Gráfico de linha

import pandas as pd

vendas = pd.DataFrame({
    "mes": ["Jan", "Fev", "Mar", "Abr", "Mai"],
    "receita": [1200, 1500, 900, 1800, 2100],
})

fig, ax = plt.subplots()
ax.plot(vendas["mes"], vendas["receita"])
plt.show()

Gráfico de linha é o melhor para mostrar evolução ao longo do tempo (meses, dias, anos).

Exemplo 2 (resolvido) — Gráfico de barras

vendas_categoria = pd.DataFrame({
    "categoria": ["Papelaria", "Acessórios", "Eletrônicos"],
    "receita_total": [3200, 1800, 4500],
})

fig, ax = plt.subplots()
ax.bar(vendas_categoria["categoria"], vendas_categoria["receita_total"])
plt.show()

Gráfico de barras é o melhor para comparar categorias entre si.

Exemplo 3 (resolvido) — Gráfico de dispersão

fig, ax = plt.subplots()
ax.scatter(vendas["receita"], [10, 14, 8, 18, 21])  # ex.: receita x nº de pedidos
plt.show()

Gráfico de dispersão (scatter) é o melhor para mostrar a relação entre duas variáveis numéricas.

Tipo de gráficoQuando usar
Linha (ax.plot)Evolução ao longo do tempo
Barra (ax.bar)Comparar categorias
Dispersão (ax.scatter)Relação entre 2 variáveis numéricas
💡 Esqueceu de exibir o gráfico no Colab? Se nada aparecer, confira se a célula termina com plt.show() (ou se o notebook está com %matplotlib inline ativo — no Colab isso já vem configurado por padrão).
✏️ Agora é sua vez: no notebook, crie um gráfico de barras com um DataFrame de sua escolha.

2. Customização, Subplots e Exportação

Exemplo 1 (resolvido) — Customizando um gráfico

fig, ax = plt.subplots()
ax.plot(vendas["mes"], vendas["receita"], color="#CC0000", marker="o", label="Receita")
ax.set_title("Receita mensal")
ax.set_xlabel("Mês")
ax.set_ylabel("Receita (R$)")
ax.legend()
plt.show()

set_title(), set_xlabel()/set_ylabel() e legend() são o mínimo para um gráfico profissional — sem eles, ninguém além de quem criou o gráfico entende o que está vendo.

Exemplo 2 (resolvido) — Subplots: vários gráficos lado a lado

fig, axs = plt.subplots(1, 2, figsize=(10, 4))  # 1 linha, 2 colunas de gráficos

axs[0].plot(vendas["mes"], vendas["receita"])
axs[0].set_title("Evolução mensal")

axs[1].bar(vendas_categoria["categoria"], vendas_categoria["receita_total"])
axs[1].set_title("Por categoria")

plt.show()

axs (no plural) é uma lista de eixos — axs[0] é o primeiro gráfico, axs[1] é o segundo, e assim por diante.

Exportação de gráficos

fig.savefig("receita_mensal.png", dpi=150)

fig.savefig() salva a figura inteira (não a tela) em um arquivo de imagem — essencial para colocar o gráfico num relatório ou apresentação.

✏️ Agora é sua vez: no notebook, crie uma figura com 2 subplots, customize com título e rótulos, e exporte com savefig().

3. Web Scraping

Web scraping é extrair dados diretamente de páginas web — útil quando o site não oferece uma API para isso.

⚠️ Antes de fazer scraping de qualquer site real: verifique o arquivo robots.txt do site e os termos de uso. Fazer muitas requisições rápidas pode sobrecarregar o servidor (e te bloquear). Por isso, todos os exemplos abaixo usam o quotes.toscrape.com — um site público criado especificamente para prática de scraping, sem problema ético em ser usado para aprender.

Exemplo 1 (resolvido) — Buscando o HTML de uma página

import requests

resposta = requests.get("http://quotes.toscrape.com/")
print(resposta.status_code)   # 200 = sucesso
print(resposta.text[:300])    # os primeiros 300 caracteres do HTML
⚠️ Veja como é um erro real do Python: se você esquecer o http:///https:// na URL, o `requests` recusa e mostra: 
requests.get("quotes.toscrape.com")
MissingSchema: Invalid URL 'quotes.toscrape.com': No scheme supplied. Perhaps you meant https://quotes.toscrape.com?

Exemplo 2 (resolvido) — Extraindo dados com BeautifulSoup

from bs4 import BeautifulSoup

sopa = BeautifulSoup(resposta.text, "html.parser")
citacoes = sopa.find_all("span", class_="text")
autores = sopa.find_all("small", class_="author")

for citacao, autor in zip(citacoes[:3], autores[:3]):
    print(autor.text, "-", citacao.text)

find_all() procura, dentro do HTML, todas as tags que combinam com o seletor (aqui, <span class="text">) — é assim que você "garimpa" a informação específica que quer, dentro de uma página inteira de HTML.

✏️ Agora é sua vez: no notebook, extraia as 5 primeiras tags <span class="text"> de quotes.toscrape.com e guarde em uma lista.

4. Funções Reutilizáveis e Introdução a Classes

Você já cria funções desde a Semana 04. Mas há situações em que uma função não basta: quando você precisa agrupar dados e comportamento juntos, e manter esses dados entre uma chamada e outra.

Exemplo 1 (resolvido) — Função de transformação reutilizável

def normalizar(coluna):
    return (coluna - coluna.min()) / (coluna.max() - coluna.min())

vendas["receita_normalizada"] = normalizar(vendas["receita"])
print(vendas)

Uma função pura assim (recebe dados, devolve dados) é o suficiente quando você só precisa transformar um valor — é o que você já fazia desde a Semana 04.

Exemplo 2 (resolvido) — Sua primeira classe

class Produto:
    def __init__(self, nome, preco):
        self.nome = nome
        self.preco = preco

    def aplicar_desconto(self, percentual):
        self.preco = self.preco * (1 - percentual / 100)

mochila = Produto("Mochila", 150.0)
mochila.aplicar_desconto(10)
print(mochila.nome, mochila.preco)
TermoO que é
class Produto:Define o "molde" — o que todo Produto vai ter/fazer
__init__Roda automaticamente ao criar um Produto novo, guardando seus dados iniciais
selfRepresenta "este Produto específico" — é como o método acessa os dados guardados nele
mochila = Produto(...)Cria um objeto — uma "cópia preenchida" do molde Produto
💡 Função x Classe: uma função recebe dados, processa, devolve um resultado — e esquece tudo depois. Uma classe guarda dados (no self) e empacota junto as ações que fazem sentido para aqueles dados — útil quando o "dado" e o "comportamento" pertencem juntos (ex.: um Produto sempre sabe seu próprio preço, e sempre sabe como aplicar desconto em si mesmo).
⚠️ Veja como é um erro real do Python: se você esquecer o self ao definir um método: 
class Produto:
    def mostrar():  # faltou o self
        print("produto")

Produto("Caderno", 12.5).mostrar()
TypeError: Produto.mostrar() takes 0 positional arguments but 1 was given
Todo método de uma classe precisa do self como primeiro parâmetro — é assim que Python passa "qual objeto" está chamando aquele método.
✏️ Agora é sua vez: no notebook, crie uma classe Cliente com nome e saldo, e um método depositar(valor) que aumenta o saldo.

5. ETL e Integração Python-SQL

ETL significa Extract (extrair) → Transform (transformar) → Load (carregar). Você já faz isso desde a Semana 04 — só não tinha esse nome:

Etapa do ETLO que você já aprendeu que se encaixa aqui
ExtractLer um arquivo (pd.read_csv) ou fazer scraping de uma página (Seção 3)
TransformLimpar e transformar com Pandas (Semana 06)
LoadSalvar o resultado — em arquivo (to_csv) ou em um banco de dados (nesta seção)

Exemplo 1 (resolvido) — Um pipeline ETL completo

vendas.to_csv("vendas_bruto.csv", index=False)         # (preparando o exemplo)

dados = pd.read_csv("vendas_bruto.csv")                 # E — Extract
dados["receita_normalizada"] = normalizar(dados["receita"])  # T — Transform
dados.to_csv("vendas_tratado.csv", index=False)         # L — Load (em arquivo)
print("Pipeline concluído.")

Exemplo 2 (resolvido) — Load num banco de dados com sqlite3

import sqlite3

conexao = sqlite3.connect("dados.db")
dados.to_sql("vendas", conexao, index=False, if_exists="replace")  # L — Load (em banco)

resultado = pd.read_sql("SELECT * FROM vendas WHERE receita > 1000", conexao)
print(resultado)

conexao.close()
💡 Por que sqlite3 e não PostgreSQL aqui? sqlite3 não precisa de servidor nem configuração — é o jeito mais simples de praticar a ideia de Python conversando com SQL. No mercado, o mesmo `pandas.to_sql()`/`pd.read_sql()` funciona com PostgreSQL (via `SQLAlchemy`), só muda a conexão.
✏️ Agora é sua vez: no notebook, monte um pipeline ETL com seus próprios dados: leia um CSV, transforme com uma função sua, e carregue numa tabela SQLite.
