1. Valores Ausentes (NaN)

Um valor ausente (NaN — "Not a Number") acontece quando um dado não foi preenchido: um sistema não capturou a informação, uma planilha ficou com célula vazia, um sensor falhou na leitura.

Exemplo 1 (resolvido) — Detectando valores ausentes

import pandas as pd
import numpy as np

vendas = pd.DataFrame({
    "produto": ["Caderno", "Lápis", "Mochila", "Borracha", "Caneta", "Caneta", "Estojo"],
    "categoria": ["Papelaria", "Papelaria", "Acessórios", "Papelaria", "Papelaria", "Papelaria", "Acessórios"],
    "preco": [12.5, 2.0, 150.0, np.nan, 3.5, 3.5, 9999.0],
    "quantidade_vendida": [30, 80, 5, 60, 45, 45, 2],
})

print(vendas.isna().sum())

isna() marca cada célula como True (é nulo) ou False (não é); .sum() soma os True de cada coluna — assim você vê de uma vez quantos nulos tem cada coluna.

⚠️ Veja como é um erro real do Python: se você passar o nome de uma coluna errada para dropna(subset=...), o Pandas recusa e mostra: 
vendas.dropna(subset=["coluna_errada"])
KeyError: ['coluna_errada']
Confira vendas.columns antes de informar o nome de uma coluna.

Exemplo 2 (resolvido) — Tratando valores ausentes: dropna() x fillna()

sem_nulos = vendas.dropna()              # remove a LINHA inteira que tem nulo
print(sem_nulos.shape)

preco_corrigido = vendas["preco"].fillna(vendas["preco"].mean())  # substitui pela média
print(preco_corrigido)
💡 Quando usar cada um? Use dropna() quando a linha sem aquele dado não serve para a análise (ex.: falta o preço, e sem preço não dá para calcular receita). Use fillna() quando faz sentido estimar um valor razoável no lugar do vazio (ex.: usar a média, ou 0, ou "não informado").
⚠️ Armadilha real: vendas["preco"].fillna(0) sozinho não altera vendas — ele só devolve uma cópia com os nulos preenchidos. Para o DataFrame original mudar de fato, reatribua: vendas["preco"] = vendas["preco"].fillna(0) (ou use inplace=True). Esquecer isso é o erro mais comum desta semana.
✏️ Agora é sua vez: no notebook, crie um DataFrame com pelo menos 1 valor nulo, detecte com isna().sum() e trate com fillna(), reatribuindo o resultado.

2. Duplicidades e Outliers

Exemplo 1 (resolvido) — Detectando e removendo duplicidades

print(vendas.duplicated())          # True na 2ª ocorrência de uma linha idêntica

vendas_sem_duplicatas = vendas.drop_duplicates()
print(vendas_sem_duplicatas.shape)
⚠️ Veja como é um erro real do Python: um erro de digitação comum é esquecer o "s" do plural: 
vendas.drop_duplicate()
AttributeError: 'DataFrame' object has no attribute 'drop_duplicate'
O método certo é drop_duplicates(), sempre no plural.

Exemplo 2 (resolvido) — Detectando outliers com describe() e IQR

print(vendas["preco"].describe())

q1 = vendas["preco"].quantile(0.25)
q3 = vendas["preco"].quantile(0.75)
iqr = q3 - q1
limite_superior = q3 + 1.5 * iqr

print(vendas[vendas["preco"] > limite_superior])

O IQR (intervalo interquartil = Q3 − Q1) mede a "dispersão típica" dos dados; valores muito acima de Q3 + 1,5×IQR são candidatos a outlier — números que provavelmente são erro de digitação (ex.: 9999.0 em vez de 99.99) ou um evento realmente fora do padrão.

💡 Ordem importa: se você calcular a média para preencher nulos (Seção 1) antes de tratar o outlier, a média já vem distorcida pelo valor errado — no dataset desta página, a média de preco com o outlier ainda dentro é 1695,08; sem ele, cai para 33,6. Trate outliers antes de usar a média/desvio padrão para qualquer outra decisão.
✏️ Agora é sua vez: no notebook, crie um DataFrame com 1 linha duplicada e 1 outlier, e use drop_duplicates() e o método IQR para identificar os dois problemas.

3. Normalização e Colunas Calculadas/Condicionais

Exemplo 1 (resolvido) — Normalização min-max

vendas["preco_normalizado"] = (
    (vendas["preco"] - vendas["preco"].min())
    / (vendas["preco"].max() - vendas["preco"].min())
)
print(vendas[["produto", "preco", "preco_normalizado"]])

Normalizar é reescalar uma coluna para o intervalo 0 a 1, mantendo as proporções entre os valores. É útil quando você precisa comparar colunas com escalas muito diferentes (ex.: preço em reais x quantidade em unidades) sem que a escala maior "domine" o resultado.

Exemplo 2 (resolvido) — Coluna calculada com transformação condicional

vendas["faixa_preco"] = np.where(vendas["preco"] > 50, "Alto", "Baixo")
print(vendas[["produto", "preco", "faixa_preco"]])
💡 Por que não um for? Você poderia percorrer linha a linha com um `for` e um `if` dentro, classificando cada preço manualmente — funcionaria, mas seria mais lento e mais código. np.where(condição, valor_se_true, valor_se_false) faz a mesma coisa numa linha só, de forma vetorizada (mesma ideia de operação vetorizada do NumPy vista na Semana 05).
✏️ Agora é sua vez: no notebook, normalize a coluna quantidade_vendida e crie uma coluna condicional "alta_demanda" com np.where().

4. Concatenação e Caso Real

Exemplo 1 (resolvido) — Combinando dois meses com pd.concat()

vendas_fevereiro = pd.DataFrame({
    "produto": ["Caderno", "Lápis"],
    "categoria": ["Papelaria", "Papelaria"],
    "preco": [12.5, 2.0],
    "quantidade_vendida": [25, 70],
})

vendas_jan_fev = pd.concat([vendas, vendas_fevereiro], ignore_index=True)
print(vendas_jan_fev.shape)

pd.concat([df1, df2]) empilha as linhas de dois DataFrames com colunas iguais — útil quando os dados de meses (ou lojas, ou turmas) diferentes chegam em arquivos separados. ignore_index=True reinicia o índice em vez de repetir os números originais de cada DataFrame.

Caso real (resolvido) — O pipeline completo de limpeza

Juntando tudo o que esta semana ensinou, na ordem certa:

vendas_csv = vendas.copy()
vendas_csv.to_csv("vendas_sujo.csv", index=False)

dados = pd.read_csv("vendas_sujo.csv")                       # 1. ler o arquivo real
dados = dados.drop_duplicates()                              # 2. remover duplicidades
q1, q3 = dados["preco"].quantile([0.25, 0.75])
limite = q3 + 1.5 * (q3 - q1)
dados = dados[dados["preco"] <= limite]                      # 3. remover outlier (já sem distorcer a média)
dados["preco"] = dados["preco"].fillna(dados["preco"].mean()) # 4. tratar nulos (com a média já confiável)
dados["faixa_preco"] = np.where(dados["preco"] > 50, "Alto", "Baixo")  # 5. transformar

print(dados)
💡 A ordem deste pipeline não é arbitrária: duplicidades e outliers são removidos antes de calcular a média para preencher nulos — assim a média usada não vem distorcida (ver tip box da Seção 2).
✏️ Agora é sua vez: no notebook, crie um segundo "mês" de dados e combine com pd.concat() antes de repetir o pipeline de limpeza.
